# Sprint 8, 9, 10 — Documentation

## Sprint 8: WhatsApp Notification

### What
WhatsApp notification system using deep link (`wa.me/{phone}?text={message}`) for customer communication.

### API
- `POST /api/admin/notifications/whatsapp` — Generate WhatsApp deep link
  - Types: `confirm`, `reminder`, `rating_request`
  - Rate limited, input sanitized
  - Logs to `notification_logs` table

### UI
- WhatsAppButton component — reusable button for sending notifications
- Admin bookings page: "WA Konfirmasi" (confirmed), "WA Minta Review" (completed)

### Flow
1. Admin clicks "WA Konfirmasi" on confirmed booking
2. API generates pre-filled WhatsApp message
3. Opens `wa.me/{phone}?text={message}` in new tab
4. Admin sends manually from WhatsApp

### Limitations
- Uses deep link, not WhatsApp Business API (no automated delivery)
- Manual send from admin's WhatsApp
- Future: integrate WhatsApp Business API for automated H-1 reminders

## Sprint 9: Dynamic Pricing Engine

### What
Dynamic pricing with multipliers based on peak hours, day type, and demand.

### API
- `GET /api/pricing/calculate?courtId=X&date=Y&startTime=Z&endTime=W`
  - Returns: `baseTotal`, `finalTotal`, `multiplier`, `multipliers[]`, `demand`
  - Multiplier breakdown with reasons

### Multipliers
| Condition | Multiplier | Description |
|-----------|-----------|-------------|
| Peak Hour (17:00-21:00) | 1.25x | Jam ramai after work |
| Early Bird (06:00-08:00) | 0.9x | Diskon pagi hari |
| Night (21:00-23:00) | 0.95x | Diskon malam hari |
| Weekend | 1.15x | Weekend surcharge |
| High Demand (>70% booked) | 1.2x | Surcharge saat ramai |
| Low Demand (<20% booked) | 0.9x | Diskon saat sepi |

### Server-side Integration
- `pricing-service.ts`: `calculateDynamicPrice()` with peak/day/demand multipliers
- BookingService.createBooking() uses dynamic pricing automatically
- Booking form shows live price preview

### UI
- Booking form: dynamic price preview with multiplier breakdown
- Shows base price (strikethrough) when multiplier ≠ 1x
- Savings/surcharge display

## Sprint 10: Referral & Gamification

### Referral System
- `GET /api/referrals?customerId=X` — list referrals, transactions, stats
- `POST /api/referrals` — generate referral code
- `PATCH /api/referrals` — apply referral code (50k referrer + 25k referee)
- DB: `referrals` table (code, status, points)

### Gamification System
- `GET /api/gamification/achievements?customerId=X` — 18 achievements
- Tiers: Bronze (100pts), Silver (250pts), Gold (500pts), Platinum (1000pts)
- Categories: bookings, spending, points, referrals, reviews

### Admin Customers
- `GET /api/admin/customers` — list all customers with loyalty tier
- `/admin/customers` page with tier stats

### UI
- `/profile/referral` — referral dashboard
- `/profile/gamification` — achievements page
